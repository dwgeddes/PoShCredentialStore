TOPIC
    about_PSCredentialStore

SHORT DESCRIPTION
    PSCredentialStore is a cross-platform PowerShell module for secure credential management using native credential stores.

LONG DESCRIPTION
    The PSCredentialStore module provides a secure way to store and retrieve credentials
    using the native credential store on each platform:
    - Windows: Windows Credential Manager
    - macOS: Keychain Services
    - Linux: Secret Service (planned)

    The module offers a unified interface across platforms through standard PowerShell commands,
    abstracting the platform-specific implementations while maintaining security best practices.

    This module requires PowerShell 7.0 or later and is designed for cross-platform compatibility.

COMMANDS
    New-StoredCredential
        Creates a new credential in the native credential store.
        
    Get-StoredCredential
        Retrieves a credential or lists all credentials from the native credential store.
        
    Set-StoredCredential
        Updates an existing credential in the native credential store.
        
    Remove-StoredCredential
        Removes a credential from the native credential store.
        
    Test-StoredCredential
        Tests if a credential exists in the native credential store and optionally validates it.

PLATFORM SUPPORT
    Windows:
        - Uses Windows Credential Manager
        - Supports metadata storage
        - Requires appropriate user permissions
    
    macOS:
        - Uses Keychain Services via 'security' command
        - Supports rich metadata and Passwords app integration
        - May require keychain unlock
    
    Linux:
        - Planned support via Secret Service API
        - Will require libsecret-tools package

EXAMPLES
    
    # Create a new credential
    $cred = Get-Credential
    New-StoredCredential -Name "MyService" -Credential $cred -Description "API Access"

    # Check if a credential exists
    if (Test-StoredCredential -Name "MyService") {
        Write-Host "Credential exists"
    }

    # Retrieve a credential
    $storedCred = Get-StoredCredential -Name "MyService"
    $username = $storedCred.UserName
    $password = $storedCred.Credential.GetNetworkCredential().Password

    # Update an existing credential with metadata
    $newCred = Get-Credential
    Set-StoredCredential -Name "MyService" -Credential $newCred -URL "https://api.example.com"

    # Remove a credential
    Remove-StoredCredential -Name "MyService"

    # List all credentials
    Get-StoredCredential

    # Test with detailed results
    Test-StoredCredential -Name "MyService" -Detailed

    # Pipeline operations
    "Service1", "Service2" | Test-StoredCredential
    
    # Bulk operations with metadata
    @("App1", "App2") | ForEach-Object {
        New-StoredCredential -Name $_ -Credential (Get-Credential -Message "Enter credentials for $_") -Application $_
    }

SECURITY CONSIDERATIONS
    - Credentials are stored using platform-native security mechanisms
    - Memory cleanup is performed after sensitive operations
    - Input validation prevents injection attacks
    - Platform-specific security features are leveraged (e.g., keychain sync on macOS)

NOTES
    This module interacts with native credential stores using platform-appropriate APIs:
    - Windows: Win32 Credential Management API
    - macOS: 'security' command-line tool accessing Keychain Services
    - All platforms: Secure memory handling and cleanup

    The module provides consistent behavior across platforms while respecting
    platform-specific capabilities and security models.
