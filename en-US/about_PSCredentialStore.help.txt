TOPIC
    about_PoShCredentialStore

SHORT DESCRIPTION
    PoShCredentialStore is a cross-platform PowerShell module for secure credential management using native credential stores.

LONG DESCRIPTION
    The PoShCredentialStore module provides a secure way to store and retrieve credentials
    using the native credential store on each platform:
    - Windows: Windows Credential Manager (via CredMan)
    - macOS: Keychain Services (via security command)
    - Linux: In-memory storage (cross-platform compatibility)

    The module offers a unified interface across platforms through standard PowerShell commands,
    abstracting the platform-specific implementations while maintaining security best practices.

    This module requires PowerShell 5.1 or later and is designed for cross-platform compatibility.

COMMANDS
    Set-PoShCredential
        Creates or updates a credential in the credential store.
        
    Get-PoShCredential
        Retrieves a credential from the credential store.
        
    Remove-PoShCredential
        Removes a credential from the credential store.
        
    Get-CredentialStoreInfo
        Gets information about the credential store implementation and platform.

PLATFORM SUPPORT
    Windows:
        - Uses Windows Credential Manager via CredMan
        - Supports persistent storage with system security
        - Requires CredMan utility for operations
    
    macOS:
        - Uses Keychain Services via 'security' command
        - Supports rich metadata and Passwords app integration
        - May require keychain unlock for access
    
    Linux and Other Platforms:
        - Uses secure in-memory storage
        - Session-based credential persistence
        - Full cross-platform compatibility

EXAMPLES
    
    # Create a new credential
    $cred = Get-Credential -Message "Enter credentials for MyService"
    Set-PoShCredential -Name "MyService" -Credential $cred

    # Check if a credential exists
    $storedCred = Get-PoShCredential -Name "MyService"
    if ($storedCred) {
        Write-Host "Credential exists"
    }

    # Retrieve and use a credential
    $cred = Get-PoShCredential -Name "MyService"
    if ($cred) {
        $username = $cred.UserName
        $password = $cred.GetNetworkCredential().Password
        # Use credentials for authentication
    }

    # Update an existing credential
    $newCred = Get-Credential -Message "Update credentials for MyService"
    Set-PoShCredential -Name "MyService" -Credential $newCred

    # Remove a credential
    Remove-PoShCredential -Name "MyService"

    # Get platform information
    $storeInfo = Get-CredentialStoreInfo
    Write-Host "Platform: $($storeInfo.Platform)"
    Write-Host "Storage: $($storeInfo.StorageMethod)"

    # Pipeline operations - check multiple credentials
    "Service1", "Service2", "Service3" | ForEach-Object {
        $cred = Get-PoShCredential -Name $_
        if ($cred) {
            Write-Host "$_ credentials found"
        } else {
            Write-Host "$_ credentials missing"
        }
    }
    
    # Bulk credential setup
    @("App1", "App2", "Database") | ForEach-Object {
        $cred = Get-Credential -Message "Enter credentials for $_"
        Set-PoShCredential -Name $_ -Credential $cred
        Write-Host "Stored credentials for $_"
    }

    # Safe credential retrieval with error handling
    try {
        $cred = Get-PoShCredential -Name "CriticalService"
        if ($cred) {
            # Use credential safely
            Invoke-RestMethod -Uri "https://api.service.com" -Credential $cred
        } else {
            Write-Warning "Credentials not found for CriticalService"
        }
    }
    catch {
        Write-Error "Failed to retrieve credentials: $($_.Exception.Message)"
    }

SECURITY CONSIDERATIONS
    - Credentials are stored using platform-native security mechanisms where available
    - Memory cleanup is performed after sensitive operations
    - Input validation prevents malformed credential names
    - Platform-specific security features are leveraged (Windows Credential Manager, macOS Keychain)
    - Cross-platform fallback provides secure in-memory storage
    - No credentials are logged or written to disk in plain text

NOTES
    This module interacts with native credential stores using platform-appropriate methods:
    - Windows: CredMan utility accessing Windows Credential Manager
    - macOS: 'security' command-line tool accessing Keychain Services
    - Other platforms: Secure in-memory credential storage

    The module provides consistent behavior across platforms while respecting
    platform-specific capabilities and security models. Always test credential
    operations in your target environment before deploying to production.

    For automation scenarios, consider platform-specific authentication methods
    and security policies that may affect credential store access.

TROUBLESHOOTING
    If credential operations fail:
    1. Check platform support with Get-CredentialStoreInfo
    2. Verify required utilities are installed (CredMan on Windows)
    3. Ensure proper permissions for credential store access
    4. Test with simple credential operations first
    5. Review error messages for platform-specific guidance
